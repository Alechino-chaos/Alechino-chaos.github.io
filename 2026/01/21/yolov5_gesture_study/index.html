基于YOLOv5框架的目标检测

总结

  • 开发环境：Windows11, NVDIA GPU, MMYOLO框架
  • PyTorch核心模型：yolov5
  • 数据集来源：https://universe.roboflow.com/

安装mmyolo

直接在https://github.com/open-mmlab/mmyolo上找安装教程。

  • conda create -n mmyolo python=3.8 pytorch==1.10.1 ...是使用 conda 创建一个新的虚拟环境，并起名 mmyolo。
  • conda activate mmyolo用来激活这个环境，进入到mmyolo里。
  • 其中git clone https://github.com/open-mmlab/mmyolo.git会把 mmyolo 的源代码下载到电脑上

训练流程

  1. 框架认知：

    • 确认使用 MMYOLO 工具箱。
    • 明确模型架构为 YOLOv5-s (Small版本)：
      • Backbone: CSPDarknet (负责特征提取)
      • Neck: PAFPN (负责多尺度特征融合)
      • Head: YOLOv5 Head (负责最终分类与回归)

  2. 配置文件：

    • 读取_base_，使配置文件基于 yolov5 框架运行。
    • 在train_dataloader中修改batch_size(每次给显卡训练几张图，考验显卡性能)和num_workers(CPU 派几个搬运工在后台读取图片)。基于我的电脑配置，分别设置为8，2较好，兼顾速度和内存，显卡安全。
    • 因为我要进行剪刀石头布的目标识别，所以修改 class_names 为 (‘rock’, ‘paper’, ‘scissors’)。
    • 在model中，有backbone, neck, bbox_head三个层次，需要把bbox_head中的num_classes改为3，对应三种类别。
    • 修改训练参数max_epochs(训练多少轮)和val_interval(每隔多少轮考试一次并生成mAP数据)。

  3. 开始训练：

    • 先激活环境:conda activate mmyolo
    • 在终端中输入python tools/train.py my_train_config.py(其中my_train_config.py就是自己的配置文件)
    • 如果之前已经训练过却因一些问题中途退出了，想继续刚才的训练，则输入python tools/train.py my_train_config.py --resume

  4. 结果评估：

    • mAP(mean Average Precision) – 平均精度
    • Recall – 召回率：能找出多少目标
    • Loss – 损失值
      • loss_cls – 分类损失
      • loss_bbox – 定位损失
      • loss_obj – 目标损失
    • loU(Intersection over Union) – 交并比(重叠度)：模型画的框和真实框重叠了多少

常见问题

  1. 内存告急(OOM风险)：在进行模型训练的时候，会占用大量的内存，这时我们要在任务管理器中结束一些任务来腾出空间。下面几个是我的电脑上占用较多的

    • Google Chrome
    • vscode
    • DTS音效处理(异常占用)

  2. 致命报错

    • 报错信息：RuntimeError: CUDA error: an illegal memory access was encountered
    • 根本原因：硬件不稳定。显存过热(没有做好散热)。
    • 解决办法：重启，做好散热，然后使用--resume指令接着跑。

  3. 格式冲突

    • YOLO格式(.txt)：归一化描述坐标，如0 0.5000 0.5000 0.2000 0.3000。在训练时，模型经常需要对图片进行缩放（比如把你 1920x1080 的图缩放到 640x640）。如果不告诉模型原图多大，它在反算坐标时就会懵圈。
    • COCO格式(.json)：用绝对值描述，如"bbox": [320, 240, 128, 192]。标准的 COCO JSON 格式里，每一张图片的信息都包含height 和 width。

    由于计算 mAP 的标准工具 pycocotools 只认 .json 格式，因此我们可以写一个转化脚本make_json.py来进行反归一化，把 labels 文件夹中所有的 .txt 文件转换成一个大的 .json 文件。
